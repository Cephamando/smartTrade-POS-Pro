1. System Overview & Roles

smartTrade-POS-Pro is a robust, multi-tier Point of Sale and management system designed for high-volume retail and hospitality environments. It ensures secure cash handling, seamless kitchen routing, and strict inventory control.

User Roles & Permissions

Role Description & Access Level
Cashier Standard POS access. Can add to tabs, charge items, and generate receipts. Cannot end shifts or authorize payouts without Manager credentials.
Kitchen / Head Chef Access to the Kitchen Display System (KDS) and Menu Builder. Can manage recipes and adjust available meal portions.
Manager Can authorize till payouts, perform X-Reads, close shifts, and manage active cashiers. Can view detailed shift audits.
Admin / Dev Full system access. Can modify system settings (white-labeling), change licensing tiers, execute system overrides, and manage user roles.

2. Point of Sale (POS) Operations

2.1 The POS Workspace

The POS workspace features an optimized layout with a fixed, floating cart panel. This ensures action buttons remain visible regardless of cart size.

  • Pagination & Search: The product grid renders up to 24 items per page for performance. Use the real-time search or category filters to locate products instantly.
  • In-Stock Only Filter: Toggle this to hide out-of-stock items. The system remembers your preference via Local Storage.
  • Cart Actions: At the bottom of the screen, you will find permanent action buttons: Charge, Split, Add to Tab, Clear, and Lost Stock.

2.2 Split Billing, Tips & Master Invoices

The system supports advanced hospitality billing requirements:

  • Even / Custom Splits: Use the Split Bill modal to divide a large bill. Pending splits automatically generate named Tabs for individual guests.
  • Guest Naming & Tips: You can edit Guest Names directly in the split modal. Dedicated input fields allow capturing specific tip amounts per guest.
  • Master Invoices: For group payments, the POS can generate a "Master Receipt" aggregating all sub-bills and tips into a single summary document.

2.3 Refunds & Tab Merging

  • Partial Refunds: Process refunds by splitting sale lines. The system automatically marks items as 'refunded', restores exact quantities to inventory, and updates the `inventory_logs`.
  • Bill Merging: Select multiple outstanding tabs from the Dashboard and merge them into a single consolidated transaction for rapid checkout.

3. Kitchen Display System (KDS) & Menu

3.1 Kitchen Workflows

Kitchen routing is strictly enforced to prevent bypasses and ensure accurate fulfillment:

Note: Cashiers cannot directly mark "Food" or "Meal" items as collected from the cart. They must be routed through the KDS.
  • Auto-Routing: Beverages and snacks bypass the kitchen and become instantly auto-ready. Food items are pushed to the KDS in 'Pending' or 'Cooking' states.
  • Notifications: Real-time polling notification badges alert staff across the POS and Dashboard when a kitchen order is marked 'Ready'.
  • Collection: Once 'Ready', cashiers must hit 'Mark Collected' to update the status to 'served' and clear the KDS screen.

3.2 Recipe Engine & Backflushing

The system utilizes a batch production and backflushing engine:

  • Recipe Builder: Chefs can define components for meals or cocktails (e.g., dynamic badges for "Made to Order" zero-stock cocktails).
  • Backflushing: Upon checkout, the system automatically deducts raw components from inventory based on the recipe blueprint.
  • Portion Control: Meals with '0' available portions automatically gray out on the POS grid to prevent overselling.

4. Shift & Financial Management

4.1 Petty Cash & Payouts

A comprehensive financial control module handles shift-based withdrawals.

  • Payouts: Cashiers can log till deductions (e.g., paying drivers or buying ice) directly from the POS modal.
  • Authorization: Crucial Security Feature: All cash withdrawals require a Manager's username and password alongside role validation. The manager's username is permanently stamped into the database audit trail.

4.2 Shift Audits & Manager Authorization

  • Live Monitoring: The Dashboard features an 'Active Employee Shifts' panel showing real-time duration and workstation status.
  • X-Reads: Running an X-Read calculates the "Expected Drawer Cash", dynamically deducting any logged petty cash expenses from total sales.
  • End Shift: Shift closure requires Manager credentials to validate variances. Shift variance reasons and expense logs are permanently tied to the authorizing manager.

5. Inventory & Reporting

smartTrade-POS-Pro tracks inventory value using retail selling prices rather than static cost prices.

  • Goods Received Voucher (GRV): Workflow is optimized via URL parameters. Clicking the "Needs Receiving" badge dynamically auto-populates the GRV stock injection form.
  • Master Reports: Accessible via the Reports Module, utilizing Flatpickr for seamless date selection. Filter by location to view:
    • Sales Performance
    • Product Velocity
    • Inventory Audit Trails (including Lost Stock and Refunds)
    • Recipe Consumption & Variances
  • Live Audit Table: A dashboard tool that allows managers to drill-down into specific shifts and fetch historical receipts instantly.

6. Admin & System Configuration

The system supports dynamic on-the-fly configuration via the `settings` database module.

  • Licensing Tiers: The UI dynamically restructures based on the active license (lite, pro, hospitality).
  • White-Labeling: Modify the business name, theme colors, and master receipt layouts via the Dev-only 'System Settings' UI without touching source code.
  • User Management: Supports dynamic role assignment with "Soft Deletion" (deactivation) to preserve historical sales data associated with departed staff.
  • Global Location Override: Admins and Devs can seamlessly switch locations from the header modal without being locked into active shifts.
